技术架构
前端语言：HTML + CSS + JavaScript（ES6 class）

渲染方式：Canvas 2D API 自绘一切元素，未使用任何第三方库

状态管理：基于内存对象维护楼层‑元素树，配合历史栈实现撤销/重做

响应式：支持桌面端和移动端，通过媒体查询 + 触控事件适配

核心功能模块
1. 楼层系统
支持多楼层（1F、2F……），可增、删、移动、重命名楼层

楼梯可以绑定目标楼层，切换显示

2. 建筑元素绘制
墙体：矩形，可任意旋转，支持智能连结

门：含铰链位置（左/右/上/下）和开合角度（0‑360°），物理旋转可视化

窗户、楼梯、遮挡物（矩形/圆形）

3. 人员单位
操作员（友方）、敌人、人质

可设置名称、呼号、职位（操作员专属）

移动端/桌面端都支持右键或长按拖出枪线（火力线）

4. 视野系统
每个人员可定义 FOV 角度（30‑180°）和探测距离（有限/无限）

视野遮挡算法：实时计算从人员出发的扇形区域，受墙体、门、窗户、遮挡物阻挡后生成精确的多边形视野区

战斗中可切换实战模式，仅显示被友方视野覆盖的敌人/人质

5. 智能辅助
网格吸附 + 元素对齐：移动时自动吸附到其他元素的角点、边缘中点、中心

撤销/重做（Ctrl+Z / Ctrl+Y）、快捷键操作、画板绘制（自由笔、矩形、圆形、直线）

6. 画板标注
自由笔刷、几何形状绘制，可改变颜色和粗细

提供区域擦除功能，仅移除画板痕迹

算法原理
1. 视野遮挡算法（核心）
目标是计算每个人的实际可见区域，考虑所有障碍物的遮挡。

射线扫描法：从视点出发，在 FOV 角度范围内均匀发射 72 条射线

对每条射线，遍历所有障碍物：

矩形障碍物（含墙体/窗户/楼梯）：使用射线‑线段求交算法，将矩形四边分解为线段，计算最近交点

旋转矩形：通过旋转矩阵将射线变换到物体局部坐标系，再求交（或直接对旋转后的四边求交）

旋转门：根据铰链位置和开合角度计算实际门板四个顶点的世界坐标，再对每条边求交

圆形障碍物：使用射线‑椭圆求交公式（实际为椭圆，宽高可不等），计算判别式求最近交点

取所有交点中距离最近的作为射线终点，连接所有终点得到遮挡后的视野多边形

绘制时填充渐变色彩，显示真实被挡区域

射线‑线段求交公式：
已知射线起点 P、方向 D，线段端点 A、B。
计算叉积判断平行，再求解参数 t（射线参数）和 u（线段参数），若满足 t>0 且 0≤u≤1 则相交。

2. 实战模式隐性逻辑
开关打开后，所有敌方/人质默认不绘制

遍历每个操作员，用其视野多边形判断每个敌方：

先计算距离是否在 FOV 范围内

再检查角度差是否在 FOV 角度内

最后调用 isLineOfSightBlocked 检测是否有障碍物遮挡

只有至少被一个操作员真实看到的敌方单位才被渲染

3. 门的物理旋转
门有铰链位置（左/右/上/下）和开合角度（0‑360°）

计算门板四顶点时，以铰链为旋转中心，用二维旋转矩阵变换门板本地坐标

不同铰链侧对应不同的旋转方向（左铰链顺时针为正，右铰链逆时针）

射线检测时动态获取门板四边用于求交，实现门对视野的动态阻挡

4. 智能对齐系统
在拖动元素时，动态计算画布上所有建筑元素的角点、边缘中点、中心点作为吸附点

当鼠标/触摸点与任一吸附点距离 < 8px 时，自动修正坐标为吸附点，并提供视觉反馈

5. 撤销/重做
任何修改都保存画布状态的 JSON 快照至 historyStack

撤销时从栈中弹出并恢复上一快照，将被撤销状态压入 redoStack 实现重做

移动端适配
触摸事件完整处理（touchstart/move/end），实现单指拖动/绘制，长按 350ms 触发枪线绘制

移动端底部提供圆形撤销/重做按钮，属性面板可滑动呼出

手势提示自动弹出
