--------------------------------------------------------------------------------
                                更新概要
--------------------------------------------------------------------------------

  本次更新引入了「楼层内多场景系统」作为核心新功能，同时完成了画布
  缩放平移、智能尺寸吸附、门4向铰链完整支持等多项功能增强，并对
  移动端交互体验进行了全面升级。另外参考了https://github.com/VoltAgent/awesome-design-md的spaceX设计了一套新UI


--------------------------------------------------------------------------------
                              ✨ 新增功能
--------------------------------------------------------------------------------

  【1】多场景系统（核心功能）
  
    每个楼层现在支持多个独立场景，场景间完全隔离存储元素与枪线。
    
    功能特性：
      · 新建场景自动复制前一场景的全部元素和枪线（含位置/颜色等）
      · 场景支持自定义名称
        - 桌面端：双击 Tab 进入内联编辑模式，回车保存，Esc 取消
        - 移动端：长按 Tab → 底部弹出操作面板 → 点击重命名
      · 删除场景
        - 桌面端：鼠标悬停显示 × 按钮 / 右键菜单删除
        - 移动端：长按 Tab → 底部弹出操作面板 → 点击删除
      · 至少保留一个场景不可删除（仅剩1个时禁用删除）
    
    数据架构升级：
      旧: Layer → elements[] / gunlines[]
      新: Layer → scenes[{id, name, elements[], gunlines[]}] + currentSceneId
    
    影响范围：
      · 撤销/重做系统完整适配多场景快照结构
      · 所有元素创建、枪线、画板工具统一走 scene 级别读写
      · 画布信息栏同步显示当前楼层的当前场景名称


  【2】画布缩放与平移（希沃白板模式）
  
    支持对画布进行自由缩放和平移操作：
    
    ┌──────────┬────────────────────────────┐
    │   操作    │         说明               │
    ├──────────┼────────────────────────────┤
    │ 鼠标滚轮  │ 以光标位置为中心缩放       │
    │ 选择+拖拽 │ 空白区域拖拽平移整个画布   │
    │ 双指捏合  │ 两指捏合/张开缩放（移动端）│
    │ 单指拖动  │ 单指拖动平移画布（移动端）  │
    └──────────┴────────────────────────────┘
    
    技术细节：
      · 缩放范围：25% ~ 500%
      · 坐标转换管线：screenToCanvas() 统一处理所有事件坐标
      · 渲染管线：ctx.translate(pan) → ctx.scale(scale) → draw → restore
      · 建筑元素预览正确应用缩放平移变换


  【3】智能尺寸吸附
  
    创建建筑元素时，宽高可自动匹配已有元素的尺寸：
      · 收集当前场景所有元素的宽高值作为吸附候选
      · 吸附阈值 8px，达到时产生"吸附"视觉反馈
      · 不再自动连接附近元素，需手动贴合才触发吸附


  【4】门4向铰链完整支持
  
    门铰链从仅左/上扩展为完整的四向支持：
      · 左铰链 / 右铰链 / 上铰链 / 下铰链
      · 四个方向均有正确的视野遮挡计算
      · 门颜色跟随元素自身属性（不再硬编码灰色）
      · 右/下铰链旋转方向已修正


  【5】移动端底部操作面板（Action Sheet）
  
    场景Tab的移动端专属交互组件：
      · 长按场景Tab 500ms 触发
      · 从屏幕底部滑出圆角面板（带遮罩层）
      · 包含三个操作按钮：
        - ✏️ 重命名 → 进入Tab内联编辑
        - 🗑️ 删除场景 → 红色警告按钮（最后1个场景时置灰）
        - 取消 → 关闭面板
      · 点击遮罩区域或取消按钮关闭
      · 滑入/滑出动画效果


  【6】SpaceX 风格 UI 主题
  
    全套 UI 从蓝绿红配色切换为 SpaceX 航空航天美学：
      · 纯黑背景（#000000）+ 光谱白文字（#f0f0fa）
      · 半透明幽灵边框/表面（Ghost Surface）
      · 大写字母铭牌风格（字间距、全大写）
      · CSS 文件从 new-styles.css 切换为 spacex-ui.css


  【7】楼层管理 UI 重设计
  
    解决原布局重叠问题，提升切换便利性：
      · 楼层卡片：透明背景 + overflow:hidden 防内容溢出
      · 操作按钮（上移/下移/删除）：默认隐藏，hover/active 时显示
      · 场景Tab：药丸形状（border-radius:10px），激活时反色高亮（白底黑字+光晕）
      · 分隔线：极低透明度（0.08），明确区分楼层名与场景区
      · 移动端：楼层列表改为横向滚动，不再截断高度


--------------------------------------------------------------------------------
                              🐛 Bug 修复
--------------------------------------------------------------------------------

  #001  场景名称无法自定义
    症状：双击场景Tab无法进入编辑模式
    原因：浏览器 click 事件在 dblclick 之前触发，
          click 中调用 switchScene() → updateLayerList() 重建DOM，
          导致 dblclick 时原 tab 引用失效
    修复：click 事件增加 220ms 延迟执行，dblclick 时 clearTimeout 取消延迟

  #002  楼层名称无法自定义
    症状：点击楼层名称后无法输入文字，立即被DOM重建打断
    原因：layer-name 的 click 事件直接调用 switchLayer() → updateLayerList() 重建，
          contentEditable 元素被替换销毁
    修复：layer-name 的 click 仅阻止冒泡不切换楼层；
          新增 dblclick 事件全选文字进入编辑；
          点击卡片其他区域才触发楼层切换
